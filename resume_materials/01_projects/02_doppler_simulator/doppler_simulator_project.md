# 毫米波雷达底层驱动与多普勒数据采集系统

## 项目简介

基于 `24GHz FMCW` 雷达平台搭建多普勒数据采集与距离-速度验证系统，重点完成雷达底层驱动适配、原始采样数据获取、帧封装输出及目标运动参数验证。

## 工作描述

主要参与下位机底层驱动和采集链路开发。基于 `AT32F435 / RKB2301` 完成雷达 `AFE / ADC` 参数配置、板级初始化、采样时序控制和基于 GPIO 模拟 SPI 时序的数据输出调试；围绕 `frame/chirp` 采样流程处理 ADC 采样完成事件，将每帧 `32 × 512` 点数据组织为原始雷达帧，并设计轻量级帧封装格式，为后续多普勒处理和距离-速度验证提供稳定数据源。

## 简历版

### 推荐版

**毫米波雷达底层驱动与多普勒数据采集系统**

基于 `24GHz FMCW` 雷达平台搭建多普勒数据采集与验证系统，完成雷达底层驱动适配、原始采样数据获取、帧封装输出及距离-速度验证。主要参与下位机底层驱动和采集链路开发，完成雷达 `AFE / ADC` 参数配置、板级初始化、采样时序控制和基于 GPIO 模拟 SPI 时序的数据输出调试；建立目标距离、速度、`IF` 频率和多普勒频率之间的映射关系，辅助分析热图峰值位置。

### 更精简版

**毫米波雷达底层驱动与多普勒数据采集系统**

基于 `AT32F435 / RKB2301` 完成 `24GHz FMCW` 雷达底层驱动和采集链路开发，负责 `AFE / ADC` 配置、板级初始化、采样时序控制和基于 GPIO 模拟 SPI 时序的数据输出调试；设计原始帧封装格式，为后续距离-速度验证提供稳定数据源。

### 面试展开版

这个项目的重点不是做一个复杂界面，而是把 `24GHz FMCW` 雷达的底层驱动和板端采集链路打通。下位机侧从 `main.c` 进入后完成扩展 SRAM、BSP、GPIO 模拟 SPI 时序和 `TLV` 初始化，启动传感器后在主循环中持续调用 `Sensor_Process`；当单个 chirp 的 ADC 采样完成后，将采样缓冲区搬运到 `rawRadarCube` 对应行，当整帧采集完成后封装帧头和数据段并输出完整原始帧。Python 侧主要承担验证工具角色，用来确认原始帧格式、采样参数和二维 FFT 后的距离-速度结果是否匹配。

## 注意事项

- 简历中不要写“下位机生成 `Range-Doppler Map`”，当前代码主线更准确的是“下位机输出原始雷达帧，Python 侧完成二维 FFT 与热图验证”。
- 简历中不要直接写 `SoftSPI`，建议写成“基于 GPIO 模拟 SPI 时序的数据输出调试”；当前工程的完整原始帧输出不是硬件 SPI 外设，`UART` 更适合作为日志或历史调试链路描述。
- 简历中不要主动写 `2Tx3Rx TDM-MIMO`、虚拟通道相位补偿或板端二维 FFT，除非后续确认有对应分支或实物代码支撑。
