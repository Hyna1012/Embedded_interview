# 基于 60GHz 毫米波雷达的多人生命体征探测

## 项目简介

基于加特兰 `CAL60S244-IB` 毫米波雷达开发平台，设计并实现多人非接触式生命体征探测系统。系统面向室内场景实现多人目标跟踪、静止目标维持及呼吸/心跳检测，围绕 `4D` 点云、扩展卡尔曼滤波、`EDACM` 解调、一阶线性补偿和 `VMD` 频谱分离完成生命体征结果输出。

## 系统链路

1. 雷达前端配置 `60GHz FMCW` 波形，完成发射、接收和基础预处理。
2. 下位机基于距离估计、速度估计、`2D CFAR`、`DBF` 等处理得到包含距离、速度、角度和信噪比的 `4D` 点云。
3. 基于扩展卡尔曼滤波的群组跟踪算法，对点云进行预测、关联、分配、更新和维持，得到多人目标轨迹。
4. 针对静止或低速目标，通过长时间 `MTI` 对消和目标维持逻辑保持目标存在性判断。
5. 根据目标距离提取相位/微动信息，经 `EDACM` 相位解调和一阶线性补偿得到生命体征微动信号。
6. 使用 `VMD` 将微动信号分解为不同频率模态，提取呼吸和心跳频率及波形。
7. 下位机输出目标状态、生命体征结果和波形数据，验证侧完成解包、绘图和状态显示。

## 工作描述

负责系统联调和嵌入式算法实现。基于 `ARC + FreeRTOS` 实现帧完成队列处理、用户任务组织和多模块数据流衔接，围绕 `4D` 点云和扩展卡尔曼滤波实现多人目标跟踪、静止目标维护及目标状态输出；针对人体胸腔微动信号完成 `EDACM` 解调、一阶线性补偿和 `VMD` 频谱分离，提取呼吸与心跳频率及波形，输出生命体征结果和波形数据。

## 差异化写法

### 1. 偏下位机 / 嵌入式实现版

**基于 60GHz 毫米波雷达的多人生命体征探测系统**

- 基于 `MetaWare / ARC` 下位机工程参与雷达用户任务开发，围绕帧完成事件、点云数据、目标跟踪结果和生命体征计算结果组织实时处理流程。
- 实现/移植长时间 `MTI` 缓冲、目标维持逻辑、`VMD` 频谱分离和呼吸/心跳频率提取等模块，适配下位机内存和周期性计算约束。
- 设计下位机到上位机的数据输出链路，支持点云、跟踪目标、维持目标、`MTI` 图和生命体征波形的分类型传输与联调。

### 2. 偏算法验证版

**基于 60GHz 毫米波雷达的多人生命体征探测系统**

- 面向多人生命体征检测场景，梳理从 `4D` 点云到目标跟踪、静目标维持、微动相位提取和频谱分离的算法链路。
- 基于扩展卡尔曼滤波完成群组跟踪逻辑，结合长时间 `MTI` 解决目标静止后点云减少导致的持续测量问题。
- 对呼吸/心跳微动信号进行 `EDACM` 解调、一阶线性补偿和 `VMD` 模态分解，实现呼吸与心跳频段的分离和估计。

### 3. 偏上位机 / 系统联调版

**基于 60GHz 毫米波雷达的多人生命体征探测系统**

- 基于 `Python + PySide6 + pyqtgraph` 开发上位机，完成串口采集、帧数据解包、点云显示、跟踪目标显示和维持目标状态展示。
- 对接下位机输出的点云、目标、`MTI`、生命体征波形和检测结果，支持多人场景下目标位置、呼吸/心跳数值及波形实时可视化。
- 通过上位机中间结果展示辅助验证下位机算法链路，提升多目标跟踪、静目标维持和生命体征检测的联调效率。

### 4. 偏通信协议 / 竞赛支撑版

**基于 60GHz 毫米波雷达的多人生命体征探测系统**

- 参与下位机与 Python 上位机之间的数据通信设计，围绕点云、跟踪目标、维持目标、波形和检测结果定义分类型数据解析流程。
- 支撑系统答辩材料、技术报告和测试结果整理，将雷达基础处理、群组跟踪、目标维持和生命体征检测链路转化为可展示的竞赛作品。
- 配合完成项目功能演示和多场景测试，系统验证最多支持 `5` 个目标跟踪，静止目标呼吸/心跳检测准确率达到 `95%+ / 90%+`。

## 和已有两个雷达项目的差异

### 对比跌倒检测项目

- 跌倒项目强调“端侧动作识别 / 轻量化模型部署”。
- 新思项目强调“多人目标跟踪 / 静止目标维持 / 呼吸心跳检测”。
- 跌倒项目的核心输出是分类结果，新思项目的核心输出是目标位置、呼吸/心跳频率和波形。

### 对比多普勒数据采集项目

- 多普勒项目强调“底层驱动适配 / 原始采样数据输出 / 距离-速度验证”。
- 新思项目强调“真实多人场景下的跟踪与生命体征检测”。
- 多普勒项目更像雷达底层采集链路验证，新思项目更像面向智能家居/看护场景的系统级应用。
