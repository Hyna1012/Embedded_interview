# MCU 外设与板级调试深挖问答

> 用途：补充 GPIO、EXTI、Timer、PWM、ADC、DMA、低功耗、复位和板级调试等高频 MCU 外设问题。

## 1. 使用原则

- 外设题不要只答功能，要答配置步骤、典型坑和排查方法。
- 如果能结合美的实习的按键、门联锁、继电器、显示、蜂鸣器，会更像真实工程经验。
- 如果能结合雷达项目的 ADC、DMA、串口、定时调度，会更贴近你的项目。

## 2. GPIO / EXTI

### Q1. GPIO 输出高低电平前需要配置哪些内容？

**直接回答：**
需要配置时钟、引脚模式、输出类型、上下拉、速度和初始电平。

**展开解释：**
很多 MCU 的 GPIO 外设默认时钟不开，直接操作寄存器可能没有效果。输出模式还要区分推挽和开漏，外部电路不同，上下拉和速度也要合理配置。

**工程补充：**
控制继电器、蜂鸣器、LED 这类输出时，建议先设置安全初始电平，再切换引脚模式，避免上电瞬间误动作。

### Q2. 推挽输出和开漏输出有什么区别？

**直接回答：**
推挽能主动输出高低电平，开漏只能主动拉低，高电平依赖上拉。

**展开解释：**
推挽适合普通数字输出；开漏适合总线共享或线与逻辑，比如 `IIC`。开漏如果没有上拉，输出高电平时就会悬空。

**工程补充：**
如果 `IIC` 波形上不去，除了代码，还要检查 SDA/SCL 是否开漏配置、上拉电阻是否存在且阻值合适。

### Q3. 按键为什么要消抖？

**直接回答：**
因为机械按键在按下或释放瞬间会产生多次电平抖动。

**展开解释：**
如果不消抖，一次按键可能被软件识别成多次触发。常见消抖方式包括延时确认、定时扫描、多次采样一致、状态机消抖。

**工程补充：**
美的电控里按键、门开关都不应该只看一次电平，尤其门联锁这种安全输入，要结合默认安全态和状态确认逻辑。

### Q4. 外部中断和轮询按键怎么选？

**直接回答：**
实时性要求高或低功耗唤醒适合外部中断，普通按键扫描适合定时轮询。

**展开解释：**
外部中断响应快，但抖动会造成频繁中断；轮询实现简单，可统一做消抖和长按、连按判断。

**工程补充：**
很多产品里按键不一定用 EXTI，而是用定时器周期扫描，因为交互逻辑通常还要处理长按、组合键、消抖和界面状态。

## 3. Timer / PWM

### Q5. 定时器定时时间怎么计算？

**直接回答：**
定时时间由定时器时钟、预分频系数和自动重装载值共同决定。

**展开解释：**
一般可以理解为：定时器计数频率 = 定时器输入时钟 / 预分频；溢出周期 = 计数周期 × 自动重装载值。不同芯片还要注意定时器总线倍频规则。

**工程补充：**
如果定时周期不对，不要只看 `PSC/ARR`，还要确认系统时钟、APB 分频和定时器实际输入时钟。

### Q6. PWM 频率和占空比怎么配置？

**直接回答：**
频率主要由定时器周期决定，占空比由比较寄存器值和周期值比例决定。

**展开解释：**
例如计数从 0 到 `ARR`，比较值 `CCR` 决定高电平持续多久，占空比约等于 `CCR / ARR`。不同 PWM 模式下有效电平可能相反。

**工程补充：**
调 PWM 时要用示波器看真实波形，确认频率、占空比、电平极性和负载动作是否符合预期。

### Q7. 输入捕获可以用来做什么？

**直接回答：**
输入捕获可以测量外部信号的频率、周期、脉宽和占空比。

**展开解释：**
定时器在检测到指定边沿时锁存当前计数值，通过两次捕获值差值就能计算时间间隔。

**工程补充：**
如果要测风扇转速、编码器脉冲或外部方波，输入捕获比软件轮询更准确。

## 4. ADC / DMA

### Q8. ADC 采样值不稳定，可能有哪些原因？

**直接回答：**
可能是输入信号噪声、采样时间太短、参考电压不稳、源阻抗过大、布局干扰或软件滤波不足。

**展开解释：**
ADC 不是只配置分辨率就完事，前端模拟信号质量、参考电压、采样保持时间和地线噪声都会影响结果。

**工程补充：**
排查时可以先测输入脚真实电压，再看参考电压和采样时间，最后用均值滤波、中值滤波或滑动窗口降低抖动。

### Q9. ADC 为什么常配合 DMA？

**直接回答：**
因为 ADC 连续采样会产生大量数据，DMA 可以自动搬运，减少 CPU 负担。

**展开解释：**
如果每个采样点都进中断，CPU 开销会很高。DMA 可以把 ADC 数据搬到数组里，等半满或满中断后再批量处理。

**工程补充：**
雷达项目里 ADC 数据链路如果采样量大，就不能靠 CPU 单点搬运，要考虑 DMA、双缓冲和帧级处理。

### Q10. 什么是双缓冲？为什么有用？

**直接回答：**
双缓冲是一个缓冲区采集数据，另一个缓冲区处理数据，二者交替使用。

**展开解释：**
这样可以避免采集和处理互相阻塞，提高连续数据流系统的吞吐能力。常见于 ADC、串口、音频、雷达原始数据采集。

**工程补充：**
要注意生产速度和消费速度匹配，如果处理速度长期慢于采集速度，双缓冲也会溢出。

## 5. 低功耗 / 复位 / 板级问题

### Q11. MCU 常见复位来源有哪些？

**直接回答：**
常见有上电复位、外部复位、看门狗复位、软件复位、低电压复位和异常复位。

**展开解释：**
很多 MCU 有复位原因寄存器，可以在启动后读取，判断上一次复位来源。

**工程补充：**
如果设备现场偶发重启，不要只猜代码跑飞，应该先读取复位标志，再结合电源波形、看门狗和异常日志分析。

### Q12. 低功耗模式要注意什么？

**直接回答：**
要注意哪些外设关闭、哪些时钟保留、哪些中断能唤醒、唤醒后系统状态如何恢复。

**展开解释：**
低功耗不是简单睡眠，进入前要保存状态、关闭无用外设；唤醒后要恢复时钟、外设和通信状态。

**工程补充：**
低功耗场景最容易出问题的是唤醒源配置、时钟恢复和外设重新初始化。

### Q13. 板子上电后完全没反应，你怎么排查？

**直接回答：**
先查供电和复位，再查时钟和下载连接，最后查启动配置和程序入口。

**展开解释：**
第一步用万用表或示波器确认电源电压是否正常；第二步看复位脚是否释放；第三步看晶振或系统时钟；第四步确认下载器能否连接芯片；最后再看程序是否跑到 `main`。

**工程补充：**
这题考的是板级 bring-up 思路，不能一上来就说“我改代码试试”。

### Q14. 程序偶发跑飞，你会怎么定位？

**直接回答：**
先确认是否复位或异常，再检查栈溢出、数组越界、指针错误、中断并发和电源干扰。

**展开解释：**
可以加异常现场打印、看 `HardFault` 栈帧、打开栈水位检测、检查高频中断和共享变量，再结合示波器看电源是否有跌落或干扰。

**工程补充：**
偶发问题通常不是单点原因，要从软件边界、时序并发和硬件稳定性三条线一起缩小范围。
