# C 语言与内存高频问答

> 用途：专门补 C 语言、指针、内存、结构体、宏和函数指针等高频基础题。

## 1. 使用原则

- C 语言题不要只背概念，要能说出“嵌入式里为什么会出问题”。
- 指针、数组、内存对齐、`volatile`、`static` 是最高频区域。
- 回答时优先用“安全性、可维护性、资源受限、硬件访问”这几个角度。

## 2. 高频问答

### Q1. `static` 在 C 语言里有什么作用？

**直接回答：**
`static` 主要有两个作用：修饰局部变量时延长生命周期，修饰全局变量或函数时限制作用域。

**展开解释：**
局部 `static` 变量存放在静态存储区，不会随着函数返回而销毁；全局 `static` 变量或函数只在当前源文件内可见，可以减少命名冲突，也能隐藏模块内部实现。

**工程补充：**
嵌入式里常用 `static` 保存模块内部状态，比如协议解析状态、滤波历史值、状态机当前状态，但要注意它不是线程安全机制。

### Q2. `const` 和宏定义有什么区别？

**直接回答：**
`const` 有类型检查，宏只是预处理阶段的文本替换。

**展开解释：**
宏在编译前展开，不关心类型，容易出现优先级、副作用和调试困难问题；`const` 变量有明确类型，编译器可以做类型检查，也更利于调试。

**工程补充：**
简单常量优先用 `const` 或 `enum`，寄存器位掩码、条件编译和跨文件配置再考虑宏。

### Q3. `volatile` 和 `const` 能不能一起用？

**直接回答：**
可以，含义是“程序不能主动修改，但值可能被外部改变”。

**展开解释：**
比如某些只读硬件状态寄存器，软件不能写它，所以是 `const`；但硬件可能随时改变它，所以需要 `volatile` 防止编译器缓存旧值。

**工程补充：**
寄存器定义里经常能看到类似 `volatile const` 的语义，本质是在同时约束“软件写权限”和“编译器优化”。

### Q4. 数组和指针到底有什么区别？

**直接回答：**
数组是一块连续存储空间，指针是保存地址的变量。

**展开解释：**
数组名在多数表达式里会退化成指向首元素的指针，但数组本身不是指针。比如 `sizeof(arr)` 得到整个数组大小，而函数参数里的 `int arr[]` 实际会退化成 `int *arr`。

**工程补充：**
数组作为函数参数传递时会丢失长度，所以嵌入式接口通常会额外传 `len`。

### Q5. 野指针和空指针有什么区别？

**直接回答：**
空指针是明确不指向任何有效对象的指针，野指针是指向未知或已失效地址的指针。

**展开解释：**
空指针可以通过判空避免访问；野指针更危险，常见来源包括局部变量地址返回、内存释放后继续使用、指针未初始化。

**工程补充：**
嵌入式里访问野指针可能直接触发 `HardFault`，所以指针初始化、释放后置空、接口判空都很重要。

### Q6. 栈、堆、全局区分别放什么？

**直接回答：**
栈放局部变量和函数调用现场，堆放动态分配内存，全局区放全局变量、静态变量和常量数据。

**展开解释：**
栈由编译器和运行时自动管理，速度快但空间有限；堆由程序手动申请释放，灵活但容易碎片化；全局区生命周期贯穿整个程序。

**工程补充：**
MCU 资源有限，大数组尽量不要放栈上；长期运行系统也要谨慎使用堆，避免碎片和泄漏。

### Q7. 什么是内存对齐？为什么结构体会有填充？

**直接回答：**
内存对齐是让变量地址满足硬件访问要求，结构体填充是编译器为了满足成员对齐而插入的空字节。

**展开解释：**
很多处理器按对齐地址访问更高效，甚至某些非对齐访问会异常。结构体成员顺序不同，最终大小可能不同。

**工程补充：**
通信协议、Flash 存储和寄存器映射结构体要特别注意对齐和大小，必要时用 `sizeof`、静态断言或按字节解析，别盲目把裸数据强转成结构体。

### Q8. 大端和小端是什么？

**直接回答：**
大端是高字节存在低地址，小端是低字节存在低地址。

**展开解释：**
例如 `0x12345678`，大端内存顺序是 `12 34 56 78`，小端是 `78 56 34 12`。不同 CPU 或协议可能采用不同字节序。

**工程补充：**
串口协议、网络协议和上位机解包时一定要明确字节序。你的 `TLV` 协议如果跨平台解析，也要在协议文档里固定大小端规则。

### Q9. 宏函数和 `inline` 函数有什么区别？

**直接回答：**
宏函数是文本替换，`inline` 函数仍然是函数，具备类型检查。

**展开解释：**
宏函数容易出现参数副作用，比如 `MAX(i++, j++)` 可能多次自增；`inline` 更安全，但是否真正内联由编译器决定。

**工程补充：**
简单寄存器位操作可以用宏，但带表达式逻辑的功能更建议用 `static inline`，既减少调用开销，也更安全。

### Q10. 函数指针有什么用？

**直接回答：**
函数指针可以把“要执行的函数”当作变量传递，实现回调和策略切换。

**展开解释：**
常见场景包括中断回调、驱动抽象层、状态机动作表、协议命令分发表。它能减少大量 `if-else` 或 `switch`。

**工程补充：**
嵌入式里用函数指针要注意空指针检查和调用链可读性，不要为了“高级”把简单逻辑绕复杂。

### Q11. `malloc` 和静态分配怎么取舍？

**直接回答：**
静态分配确定性更强，`malloc` 更灵活但风险更高。

**展开解释：**
静态分配在编译或启动时就确定内存占用，不容易出现运行时申请失败；`malloc` 可以按需申请，但可能带来碎片、泄漏和不确定耗时。

**工程补充：**
MCU 长期运行项目里，核心缓冲区更推荐静态分配，比如 ADC 缓冲、串口环形缓冲、协议解析缓冲。动态内存如果要用，最好限制在初始化阶段。

### Q12. 什么是内存泄漏？

**直接回答：**
内存泄漏是申请的内存不再使用后没有释放，导致可用内存越来越少。

**展开解释：**
在有堆的系统里，如果反复 `malloc` 但忘记 `free`，长期运行后可能申请失败，甚至造成系统异常。

**工程补充：**
嵌入式设备通常要求长时间稳定运行，所以内存泄漏比 PC 小工具更危险。面试里可以说：“我会尽量避免在循环任务中频繁申请释放内存。”

### Q13. `strcpy`、`strncpy`、`memcpy` 使用时要注意什么？

**直接回答：**
核心是长度边界和结束符问题。

**展开解释：**
`strcpy` 不检查目标缓冲区大小，容易溢出；`strncpy` 不一定自动补 `\0`；`memcpy` 按字节复制，不关心字符串结束符，也不处理重叠区域。

**工程补充：**
协议解析、串口命令和日志字符串处理时，必须明确缓冲区大小，不能相信外部输入长度天然合法。

### Q14. `struct` 适合直接用来解析通信协议吗？

**直接回答：**
不一定，直接强转结构体解析协议有风险。

**展开解释：**
结构体可能有内存对齐和填充，不同编译器、不同平台的布局可能不同；同时还要考虑大小端和字段长度。

**工程补充：**
跨平台协议更稳的方式是按字节解析，明确每个字段的偏移、长度和字节序。你的 `TLV` 协议就应该固定字段格式，而不是依赖结构体裸布局。

### Q15. `static` 全局变量和普通全局变量有什么区别？

**直接回答：**
`static` 全局变量只在当前源文件可见，普通全局变量可以被其他文件通过 `extern` 访问。

**展开解释：**
`static` 限制了符号的链接范围，有利于封装模块内部状态，减少外部误用和命名冲突。

**工程补充：**
驱动模块内部状态、协议解析上下文、状态机当前状态，如果不希望外部直接访问，可以定义成 `static`，再通过函数接口操作。

### Q16. 局部变量未初始化会有什么问题？

**直接回答：**
局部自动变量未初始化时值不确定，使用它会导致随机行为。

**展开解释：**
栈上的局部变量不会默认清零，里面可能是之前函数调用留下的残值。全局变量和静态变量才会默认初始化为 0。

**工程补充：**
如果程序 Debug 偶尔正常、Release 偶尔异常，未初始化变量是重点怀疑对象之一。

### Q17. 为什么要少用魔法数字？

**直接回答：**
魔法数字可读性差，也不利于维护和修改。

**展开解释：**
比如代码里直接写 `0x20`、`128`、`3`，过一段时间很难知道它代表寄存器位、缓冲区长度还是状态编号。

**工程补充：**
嵌入式里建议用宏、枚举或 `const` 给关键参数命名，比如帧头、最大包长、状态编号、寄存器位掩码。

### Q18. C 语言里 `++i` 和 `i++` 有什么区别？

**直接回答：**
`++i` 是先自增再使用，`i++` 是先使用再自增。

**展开解释：**
单独作为一条语句时通常效果一样；放在表达式或函数参数里时顺序差异会影响结果。

**工程补充：**
不要写复杂自增表达式，比如 `arr[i++] = i;` 这类代码可读性差，还可能涉及未定义或易误解行为。

### Q19. 什么是未定义行为？

**直接回答：**
未定义行为是 C 标准没有规定结果的行为，程序可能表现为任何结果。

**展开解释：**
典型例子包括数组越界、使用野指针、有符号整数溢出、修改字符串常量、同一表达式里多次无序修改变量。

**工程补充：**
嵌入式里未定义行为可能在低优化等级下看似正常，高优化后暴露问题，所以代码要避免依赖“刚好能跑”的行为。

### Q20. 面试官问 C 语言基础，你怎么体现嵌入式意识？

**推荐回答：**
“我会把 C 语言问题和嵌入式场景联系起来。比如 `volatile` 对应寄存器、中断和 DMA 状态；数组和指针对应协议解析和缓冲区边界；`static` 对应模块内部状态封装；内存对齐对应结构体、通信协议和寄存器映射。对我来说，C 语言不只是语法，而是直接影响系统稳定性。”
