# 通信总线深挖面试问答

> 用途：补充 `UART / IIC / SPI / CAN / RS-485` 在真实面试中常见的追问。
>
> 这份文件偏“面试追问版”，和通信协议专题知识整理互补。

## 1. 使用原则

- 总线题先说核心特性，再说工程选型，再说调试方法。
- 遇到具体问题要按“物理层波形 - 协议配置 - 软件收发 - 上层解析”分层排查。
- 你项目里最容易桥接的是 `UART + TLV + 上位机解包`。

## 2. UART

### Q1. UART 为什么不需要时钟线？

**直接回答：**
因为 UART 是异步通信，收发双方通过约定波特率来同步采样。

**展开解释：**
UART 帧里有起始位，接收端检测到起始位后，按照配置好的波特率在合适时刻采样后续数据位、校验位和停止位。

**工程补充：**
所以收发双方波特率、数据位、停止位和校验位必须一致，否则就会乱码或帧错误。

### Q2. UART 乱码可能是什么原因？

**直接回答：**
常见原因是波特率不一致、时钟不准、数据格式不一致、接地问题、电平标准不匹配或线序接反。

**展开解释：**
比如 MCU 是 TTL 电平，PC 端如果走 RS-232 电平需要转换芯片；如果系统时钟配置错，波特率也会偏。

**工程补充：**
排查时先确认共地、电平、TX/RX 交叉、波特率和串口参数，再用逻辑分析仪看真实帧格式。

### Q3. UART 用中断、DMA、轮询分别适合什么场景？

**直接回答：**
轮询适合低频简单收发，中断适合中等数据量，DMA 适合高速或连续数据流。

**展开解释：**
轮询简单但浪费 CPU；中断响应及时但高频时开销大；DMA 能自动搬运数据，配合空闲中断适合不定长帧接收。

**工程补充：**
雷达项目的上位机数据回传量较大，如果数据持续输出，就要优先考虑 DMA、环形缓冲和明确帧格式。

### Q4. 串口协议为什么要有帧头和长度？

**直接回答：**
因为 UART 是字节流，帧头用于找到一帧开始，长度用于确定一帧边界。

**展开解释：**
没有帧头和长度，接收端丢一个字节后可能一直错位解析。长度字段可以让接收端知道接下来应该读多少数据。

**工程补充：**
你的 `TLV` 协议可以说：帧头解决同步，`Type` 解决数据类型，`Length` 解决边界，`Value` 承载真实数据。

## 3. IIC

### Q5. IIC 为什么需要上拉电阻？

**直接回答：**
因为 IIC 通常是开漏输出，只能主动拉低，释放后靠上拉电阻变高。

**展开解释：**
这种设计允许多个设备共享总线，不同设备都可以拉低总线，不会出现一个输出高、另一个输出低造成强冲突。

**工程补充：**
上拉过大上升沿慢，速率上不去；上拉过小功耗大、拉低电流大，所以要结合总线电容和速率选择。

### Q6. IIC 总线卡死怎么办？

**直接回答：**
常见做法是尝试手动拉 SCL 产生几个时钟，把从机释放 SDA，然后重新初始化 IIC 外设。

**展开解释：**
总线卡死可能是通信中断在某个状态，从机一直拉住 SDA。主机可以通过 GPIO 模拟时钟让从机把剩余位吐完，再发停止条件恢复。

**工程补充：**
产品里如果 IIC 连接关键器件，最好设计总线恢复机制，而不是卡死后只能重启。

### Q7. IIC 的 7 位地址和 8 位地址为什么容易混？

**直接回答：**
因为芯片手册有的写 7 位地址，有的写左移后包含读写位的 8 位地址。

**展开解释：**
IIC 实际传输地址字节时，前 7 位是地址，最低位是读写位。驱动库有的要求传 7 位地址，有的要求传左移后的地址。

**工程补充：**
如果 IIC 一直没有 ACK，要优先确认地址传参格式，不要只怀疑硬件坏了。

## 4. SPI

### Q8. SPI 为什么速度通常比 IIC 快？

**直接回答：**
SPI 协议更简单，有独立时钟、数据线和片选线，通常没有 IIC 那样的地址和 ACK 开销。

**展开解释：**
SPI 可以全双工，时序直接，适合高速 ADC、Flash、屏幕、雷达前端这类数据量较大的外设。

**工程补充：**
代价是线更多，多从机时片选管理更复杂，板级布线和信号完整性也要注意。

### Q9. SPI 片选信号要注意什么？

**直接回答：**
要注意片选有效电平、建立保持时间、是否允许连续传输、多个从机不能同时选中。

**展开解释：**
有些设备要求一整个命令期间 CS 保持有效，如果中途拉高，内部状态机会重置。也有设备对 CS 到时钟的延迟有要求。

**工程补充：**
如果 SPI 读写偶发错误，除了模式和频率，还要用逻辑分析仪看 CS 时序。

### Q10. SPI 全双工是什么意思？

**直接回答：**
全双工表示发送和接收可以同时进行。

**展开解释：**
SPI 主机发出时钟时，MOSI 输出数据，MISO 同时输入数据。即使只是读数据，主机也通常要发送 dummy byte 来产生时钟。

**工程补充：**
很多新手会问“为什么读 SPI 还要发数据”，本质是 SPI 的时钟由主机发送动作产生。

## 5. CAN / RS-485

### Q11. CAN 和 UART 最大区别是什么？

**直接回答：**
CAN 是多主总线，有仲裁、错误检测和消息 ID；UART 更像点对点异步串口。

**展开解释：**
CAN 不按设备地址通信，而是按消息 ID 过滤；多个节点可以挂在总线上，冲突时通过 ID 仲裁决定谁先发。

**工程补充：**
汽车、工业控制里常用 CAN，是因为它抗干扰强、错误处理机制完善、适合多节点实时通信。

### Q12. CAN 为什么需要终端电阻？

**直接回答：**
终端电阻用于匹配总线阻抗，减少信号反射。

**展开解释：**
高速差分总线如果两端不匹配，信号会反射，导致波形畸变和通信错误。常见 CAN 总线两端各接 `120Ω`。

**工程补充：**
CAN 通信不稳定时，要检查波特率、终端电阻、总线拓扑、共地和差分线连接。

### Q13. RS-485 和 UART 是什么关系？

**直接回答：**
UART 是通信控制器/协议格式，RS-485 是物理层电气标准。

**展开解释：**
很多系统用 UART 控制器产生串口数据，再通过 RS-485 收发器转换成差分信号，提高传输距离和抗干扰能力。

**工程补充：**
RS-485 半双工通信要控制收发方向，如果方向切换时机不对，会出现发不出去或收不到。

### Q14. 你会怎么选 `UART / IIC / SPI / CAN / RS-485`？

**直接回答：**
看距离、速率、节点数量、可靠性和硬件资源。

**展开解释：**
板内低速多设备可用 IIC；板内高速外设可用 SPI；简单点对点调试和日志可用 UART；车载或工业多节点可用 CAN；远距离抗干扰串行通信可用 RS-485。

**工程补充：**
面试里不要说哪个协议绝对更好，应该说“按场景选型”。
