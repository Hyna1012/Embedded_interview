# C 语言进阶与嵌入式代码质量问答

> 用途：补充 C 语言更贴近真实面试的追问，包括指针修饰、结构体、位操作、回调、可重入、内存安全和代码质量。

## 1. 使用原则

- C 语言进阶题要尽量结合嵌入式场景：寄存器、协议解析、状态机、中断、内存受限。
- 面试官问 C，经常不是考语法，而是考你写出的代码是否安全、可维护、可调试。

## 2. 指针与类型

### Q1. `const int *p`、`int const *p`、`int * const p` 有什么区别？

**直接回答：**
`const int *p` 和 `int const *p` 表示指向常量的指针，不能通过 `p` 改值；`int * const p` 表示指针本身是常量，不能改指向。

**展开解释：**
看 `const` 修饰谁，靠近 `*` 左边通常修饰指向的数据，靠近变量名则修饰指针本身。

**工程补充：**
驱动接口里如果不希望函数修改输入缓冲区，参数应写成 `const uint8_t *buf`，这能表达接口意图，也能减少误改。

### Q2. `typedef` 和 `#define` 定义类型有什么区别？

**直接回答：**
`typedef` 是给类型起别名，`#define` 是文本替换。

**展开解释：**
例如 `typedef int* PINT;` 后 `PINT a, b;` 中 `a` 和 `b` 都是指针；但 `#define PINT int*` 后 `PINT a, b;` 只有 `a` 是指针，`b` 是普通 `int`。

**工程补充：**
类型别名优先用 `typedef`，宏更适合常量、条件编译和简单替换。

### Q3. `enum` 有什么用？

**直接回答：**
`enum` 用来定义一组具名整数常量，让状态或选项更可读。

**展开解释：**
相比直接写数字，`enum` 能表达含义，比如系统状态、错误码、事件类型、协议命令类型。

**工程补充：**
状态机里强烈建议用 `enum` 表示状态，不要用 `0/1/2` 这种魔法数字。

### Q4. `union` 有什么用？

**直接回答：**
`union` 的多个成员共享同一块内存，适合节省空间或按不同方式解释同一段数据。

**展开解释：**
它常用于协议解析、寄存器视图、数据类型转换等场景。但由于成员共享内存，写一个成员再读另一个成员要注意可移植性和字节序。

**工程补充：**
嵌入式里 `union` 很有用，但协议解析不要过度依赖裸强转，跨平台时更稳的是按字节显式组包解包。

## 3. 位操作与寄存器

### Q5. 如何设置、清除、翻转和读取某一位？

**直接回答：**
设置用或，清除用与取反，翻转用异或，读取用与掩码。

**展开解释：**
示例：

```c
reg |= (1U << n);       /* set */
reg &= ~(1U << n);      /* clear */
reg ^= (1U << n);       /* toggle */
bit = (reg >> n) & 1U;  /* read */
```

**工程补充：**
寄存器操作要注意读改写风险。如果寄存器某些位写 1 清除，不能随便整寄存器读改写。

### Q6. 什么是位掩码？

**直接回答：**
位掩码是用某些位为 1 的数来选择或操作目标位。

**展开解释：**
比如 `0x0F` 可以选中低 4 位，`BIT(3)` 可以选中第 3 位。位掩码让寄存器和协议字段操作更清晰。

**工程补充：**
建议把寄存器位定义成有意义的宏或枚举，不要在代码里散落魔法数字。

### Q7. 为什么寄存器通常定义成 `volatile`？

**直接回答：**
因为寄存器值可能被硬件随时改变，不能让编译器缓存或优化访问。

**展开解释：**
如果不加 `volatile`，编译器可能认为两次读取结果一样，从而省略第二次读取。但硬件状态寄存器每次读都可能不同。

**工程补充：**
硬件寄存器、ISR 共享标志、DMA 状态变量都是 `volatile` 高频场景。

## 4. 函数、回调与模块设计

### Q8. 什么是回调函数？

**直接回答：**
回调函数是把函数指针传给模块，在特定事件发生时由模块反过来调用。

**展开解释：**
比如串口收到一帧数据后调用用户注册的处理函数，按键事件触发后调用应用层函数。

**工程补充：**
回调能解耦驱动层和业务层，但要注意回调执行上下文。如果回调在中断里执行，就不能做耗时操作。

### Q9. 什么是可重入函数？

**直接回答：**
可重入函数是可以被多个执行流同时调用而不会互相破坏状态的函数。

**展开解释：**
如果函数依赖静态局部变量、全局变量或共享缓冲区，并且没有保护，就通常不可重入。

**工程补充：**
中断和任务都可能调用的函数要特别注意可重入性，必要时使用局部变量、传入缓冲区或加锁保护。

### Q10. 什么是模块化设计？

**直接回答：**
模块化设计是把功能按职责拆分，每个模块暴露清晰接口，隐藏内部实现。

**展开解释：**
例如串口驱动只负责收发，协议层负责帧解析，业务层负责具体命令处理。模块之间通过接口交互，而不是互相访问内部变量。

**工程补充：**
你的 `TLV` 项目可以按“串口底层 - 帧同步 - TLV 解析 - 数据分发 - 上位机显示”来讲模块边界。

## 5. 内存安全与代码质量

### Q11. 什么是缓冲区溢出？

**直接回答：**
缓冲区溢出是写入数据超过了缓冲区边界，破坏了相邻内存。

**展开解释：**
常见于数组越界、字符串拷贝不检查长度、串口接收不限制帧长。它可能导致变量被覆盖、程序跑飞或安全漏洞。

**工程补充：**
协议解析必须检查长度字段，不能收到多长就写多长。尤其串口帧要先判断最大长度。

### Q12. `memcpy` 和 `memmove` 有什么区别？

**直接回答：**
`memcpy` 不保证处理重叠内存，`memmove` 可以安全处理重叠区域。

**展开解释：**
如果源地址和目标地址重叠，`memcpy` 结果可能不确定；`memmove` 会选择安全拷贝方向。

**工程补充：**
环形缓冲、协议缓存移动数据时，如果区域可能重叠，用 `memmove` 更稳。

### Q13. 为什么嵌入式里不建议递归？

**直接回答：**
因为递归会持续消耗栈空间，嵌入式栈通常有限，容易栈溢出。

**展开解释：**
递归深度如果不可控，就很难保证最坏情况下的栈占用。实时系统也不喜欢不可预测的调用深度和耗时。

**工程补充：**
嵌入式里更常用循环或显式栈结构替代递归，尤其在长期运行系统中。

### Q14. 你写 C 代码时怎么减少 Bug？

**直接回答：**
我会从接口边界、长度检查、初始化、错误返回和模块化几个方面控制风险。

**展开解释：**
具体包括：指针先判空，数组操作带长度，变量初始化，错误码明确返回，模块内部状态不随便暴露，关键逻辑加日志或断言。

**工程补充：**
面试里这题很适合回答成工程习惯：“我不只追求功能跑通，也会关注边界条件和后续可维护性。”
